機械処理によるデータ生成

漢字の熟語からデータ生成

KJ_form2などでハングル→日本語変換かけると同音異義語の問題で
以下のような出かたをする事が多々あります。

주행거리 → [走行,舟行][距離,通り]

주행や거리は登録しているが、주행거리は登録されていないためです。
しかし逆方向の変換は一意にハングルが決まります。

走行距離 → 주행거리

というわけでJFなどいろいろな日本語文書を持ってきて、漢字熟語を取り出し
KJ_dictに未登録でハングルに自動変換可能なものをプログラムで生成させてみました。
これで何千語か作れました。


活用形を自動生成

動詞と形容詞に関して、原型から活用形や熟語を自動生成させてみました。
これが1万語から2万語の間くらい。

京都橘大学のデータの中にある動詞や形容詞はほとんど原型だけなので、
そのプログラムに食わせれば1個が50個くらいに増えます。
すこしずつデータ移しています。

そのプログラム、あっとるんか、という問題はありますが。 (^^;
# いちおう、データをピックアップし、韓国の友人2名にチェックしてもらってます。

郵便データと自動生成

http://www.post.japanpost.jp/ や
http://www.koreapost.go.kr/ に地名と郵便番号の対応データが公開されています。
この地名データからハングル・カタカナ対応付けを自動生成しています。

日本の地名→読み仮名からハングルを自動生成し対応付ける。
韓国の地名→ハングルからカタカナを自動生成し対応付ける。

自作のスクリプトなので、相互変換は完全でないかもしれません。
地名データのライセンスは明記されていないのですが、
自由に使用可と書いてあるので、取り込んでみました。
KJ_form2で新聞記事などを食わせる時に効果あります。
