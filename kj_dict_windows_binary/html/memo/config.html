設定ファイルの名称変更 「KJ_form.config」→「KJ_dictform_setting.xml」

Ver 2.0.0.0から，開発ツールをVisual C# 2008 Express ＋ .NET Framework 3.5に変えたのですが，
KJ_form.configの読み込みで落ちるようになってしまいました。

message="構成システムを初期化できませんでした。"
innerException=System.Configuration.ConfigurationErrorsException

アプリケーション構成ファイルのファイル名が
「*.exe.config が存在しなければ、*.config を読みに行く」
という動作に変更されたようです。

そのため，KJ_form.configをアプリケーション構成ファイルとして扱いに行き，
落ちてしまう。

そのため，設定ファイルの名称を変更しました。
