韓国IMEに対する動作不良

・韓国IMEに対する動作不良対策のため，入力エリアをRichTextBoxからTextBoxに変更しました。

私は気づいてもいない，興味深い問題だったのでここで紹介します
これは韓国IMEだけで起きる現象です。

「사랑」という単語を入力する場合、こんな順番で入力されるのが正解。 

(ㅅ)
(사)
(살)
사(라)
사랑


KJ_form.exe, KJ_form2.exe では入力エリアをrichTextBoxで作成していますが，
TextChangeイベントで
String temp = richTextBox.Text;　で持ってきた場合
文字の組み合わせの状態がおかしくなります

「RichTextBox」の場合；
(ㅅ)
(사)
(살)
사(라)　→　TextChanged → String temp = richTextBox.Text; → 사라
사라ㅇ　→　TextChanged → String temp = richTextBox.Text; → 사라ㅇ

これはTextBoxであると問題ありません。
そのため，入力エリアをRichTextBoxからTextBoxに変更しました。

入力エリアをRichTextBoxで定義していたのは，TextBoxだと日本語Windows98上で
ハングルが表示できなかったためです。
しかし，そろそろWindows98を対象としなくてよいだろうと考えますので，
韓国語IMEの問題対策を優先させます
